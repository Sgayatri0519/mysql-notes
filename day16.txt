What Is a View?
A view is a virtual table in MySQL, built from the result of a stored SELECT statement. It looks and behaves like a regular table when you query it — you can SELECT from it, filter it, and join it — but it does not store any data of its own. Every time a view is queried, MySQL runs its underlying SELECT statement against the real ("base") tables and returns the result.
Think of a view as a saved, reusable query with a name. Instead of typing out a long JOIN every time, you create the view once, then simply run: SELECT * FROM view_name;

 Why Do We Need Views?
*Simplify complex queries: Hide multi-table JOINs, subqueries, and calculations behind a single, simple table-like name.
* Reuse query logic: Write a complex SELECT once and reuse it everywhere — reports, applications, dashboards — without copy-pasting SQL.
*Data security: Expose only specific columns or rows to certain users, without granting them access to the full base table.
*Different perspectives of the same data: Different departments can have their own tailored 'view' of the same underlying tables.
*Logical data independence: If the underlying table structure changes slightly, you can often adjust just the view definition and keep all dependent queries working unchanged.
 *Simplified reporting: Pre-package aggregations (totals, averages, counts) into a view so business users can query them without knowing SQL joins or GROUP BY.


Advantages of Views
●Security: restrict access to sensitive columns (e.g. salary) or sensitive rows (e.g. only a user's own department).
●Simplicity: end users and applications interact with a simple, named virtual table instead of complex SQL.
●Reusability: one view definition can be reused across many reports, applications, and ad-hoc queries.
●Consistency: business logic (calculations, filters) lives in one place, so results are always consistent.
●Maintainability: a business-rule change only needs to be made once, inside the view.
●No extra storage cost: a standard view stores no data — only its query definition.

-->Creating Views
->Basic Syntax:
CREATE VIEW view_name AS
SELECT columns
FROM tables
WHERE conditions;

->Altering Views
Syntax for ALTER VIEW:
ALTER VIEW view_name AS
SELECT new_columns
FROM new_tables
WHERE new_conditions;

Note: Not all database systems support ALTER VIEW. Some require DROP and CREATE.

->Dropping Views
Basic Syntax:
DROP VIEW view_name;
DROP VIEW IF EXISTS view_name;  -- Safe drop

mysql> CREATE VIEW vw_department_salary_summary AS
    -> SELECT d.dept_name,
    ->    	COUNT(e.emp_id)   AS total_employees,
    ->    	SUM(e.salary) 	AS total_salary,
    ->    	AVG(e.salary) 	AS average_salary
    -> FROM employees e
    -> JOIN departments d ON e.dept_id = d.dept_id
    -> GROUP BY d.dept_name;
